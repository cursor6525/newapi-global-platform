# 🇪🇺 GDPR (通用数据保护条例) 合规检查清单

> **适用范围**: 处理欧盟/EEA 用户数据的业务节点 (如 `eu` 区域)。

## ✅ 数据本地化与传输
- [ ] **数据不出境**: 欧盟用户的个人信息 (PII) 必须存储在 `eu` 区域的数据库中 (`100.64.3.10`)。
- [ ] **跨境传输限制**: 禁止将欧盟用户数据同步到 `cn`、`us` 或 `ap` 区域。
- [ ] **加密传输**: 所有跨区域 (如管理节点同步配置) 通信必须使用 TLS 1.3 或 WireGuard 加密。
- [ ] **日志脱敏**: 业务日志中不得包含未脱敏的 IP 地址、邮箱或手机号，除非有明确合法依据。

## ✅ 用户权利保障
- [ ] **被遗忘权**: 提供一键删除功能，执行后必须物理删除数据库中用户的所有记录，不留备份残留。
- [ ] **数据可携权**: 支持导出用户数据为 JSON/CSV 格式，且仅包含该用户自身数据。
- [ ] **知情同意**: Cookie 收集或数据追踪前必须获得用户明确同意。

## ✅ 安全与审计
- [ ] **DPO 任命**: 如果大规模处理敏感数据，需指定数据保护官 (DPO) 或联系人。
- [ ] **违规通报**: 发生数据泄露事件时，必须在 72 小时内通知监管机构。
- [ ] **定期审计**: 每半年执行一次 `scripts/audit-compliance.sh eu` 并归档报告。

## 🚨 违规风险
- **最高罚款**: 2000 万欧元或全球年营业额的 4% (取高者)。
- **业务影响**: 可能被禁止在欧盟境内开展业务。
