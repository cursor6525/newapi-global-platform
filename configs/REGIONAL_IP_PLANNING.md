# 🌐 NetBird 地域 IP 分配标准规划

## 核心原则
- **网段分配**: 统一使用 `100.64.0.0/10` (RFC 6598 共享地址空间)，避免与私有网段冲突。
- **分配格式**: `100.64.{区域ID}.{主机位}`。
- **区域隔离**: 每个国家/地区分配独立的 `/24` 网段，禁止跨区直接路由业务数据。
- **固定约定**: 
  - `.1` - `.9`: 保留给基础设施 (Master/Edge/Relay)。
  - `.10`: MySQL 主库。
  - `.11`: MySQL 从库。
  - `.20`: Redis 主节点。
  - `.100`: 区域 NewAPI 业务 VIP (K8s Service)。
  - `.255`: 广播/保留。

## 扩容规则
1. **新增国家**: 从 IP 池中分配新的 `/24` 网段 (如 `100.64.5.0/24` 分配给 `fr` 法国)。
2. **ACL 同步**: 分配 IP 后，必须更新 `manifests/netbird/acl-policy.json` 并重启 NetBird 同步。
3. **K8s 亲和性**: 所有新节点必须标记 `region=xx`，NewAPI Pod 必须配置 `nodeAffinity` 锁定该网段。

## 审计要求
- 每季度核对 NetBird 实际分配 IP 与本规划表。
- 任何跨区 IP 访问尝试必须被 NetBird ACL 拦截并记录日志。
